about

The big question — how does it work?

StudentVUE has created a database which developers are granted access to through an API (application programming interface).

We can then take a user's login credentials, send them to StudentVUE, and if they match the credentials that they have stored, it grants the developer access to the user's data.

At no point during this process are the user credentials exposed to the developer for extraction or exploitation.

Ok, but why?

We're tired of the laggy website and slow loading times. The buggy app that seems to break every once-in-a-while. We want a reliable, fast, and practical interface.
